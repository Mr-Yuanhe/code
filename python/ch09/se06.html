9.6 继承和多态

继承

面向对象编程的一大优点是对代码的重用（Reuse），重用的一种实现方法就是通过继承（Inheritance）机制。继承最好是想象成在类之间实现类型与子类型（Type and Subtype） 关系的工具。

在之前的例子中，学校里的老师和学生可以抽象成两个类，有一些特征是他们都具有的，例如姓名、年龄和性别。另外一些特征是他们各自独有的，例如教师的薪水、课程与假期，学生的成绩和学费。

学生类
属性：姓名、性别、年龄、学历、手机号码、个人账户余额等
功能：提出报名，缴费

教师类
属性；姓名、性别、年龄、学历、手机号码、员工编号等
功能：登记报名信息，分配教室等

我们可以为老师和学生创建两个独立的类，并对它们进行处理。但增添上面所说的共有特征就意味着将其添加进两个独立的类。这很快就会使程序变得笨重。修改起来也容易遗漏从而产生错误。

更好的方法是创建一个公共类叫做SchoolMember，然后让老师类和学生类从这个类中继承（Inherit），也就是说他们将成为这一类型（类）的子类型，而我们就可以向这些子类型中添加某些该类独有的特征。

SchoolMember类
属性：姓名、性别、年龄等
功能：活着

这种方法有诸多优点。如果我们增加或修改了SchoolMember的任何功能，它将自动反映在子类型中。举个例子，可以通过简单地向 SchoolMember类进行操作，来为所有老师与学生添加一条新的ID卡字段。而对某一子类型作出的改动并不会影响到其它子类型。

同时还需要注意的是我们重用父类的代码，但我们不需要再在其它类中重复它们，当我们使用独立类型时才会必要地重复这些代码。

在上文设想的情况中，SchoolMember类会被称作基类（Base Class）或是超类（Superclass）或是父类。Teacher 和Student类会被称作派生类（Derived Classes） 或是子类（Subclass）。

Python中使用继承，在定义类时需要在类后面跟一个包含基类名称的元组：

# 公共类，基类
class SchoolMember:
    '''代表任何学校里的成员。'''

    def __init__(self, name, age):
        self.name = name
        self.age = age
        print('(初始化 SchoolMember: {})'.format(self.name))


# 老师类，从SchoolMember继承
class Teacher(SchoolMember):
    '''代表一位老师。'''

    def __init__(self, name, age, salary):
        SchoolMember.__init__(self, name, age)
        self.salary = salary 
        print('(初始化 Teacher: {})'.format(self.name))

基类的__init__方法是通过self变量被显式调用的，因此我们可以初始化对象的基类部分。

因为在Teacher和Student子类中定义了__init__方法，Python不会自动调用基类SchoolMember的构造函数，必须自己显式地调用它。

相反，如果没有在一个子类中定义一个__init__方法，Python将会自动调用基类的构造函数。

这时可以通过在方法名前面加上基类名作为前缀，再传入self和其余变量，来调用基类的方法：9.5-继承.py

# 继承

# 公共类，基类
class SchoolMember:
    '''代表任何学校里的成员。'''

    def __init__(self, name, age):
        self.name = name
        self.age = age
        print('(初始化 SchoolMember: {})'.format(self.name))

    def tell(self):
        '''输出对象信息'''
        print('Name:{}\tAge:{}\t'.format(self.name, self.age), end='\t')


# 老师类，从SchoolMember继承
class Teacher(SchoolMember):
    '''代表一位老师。'''

    def __init__(self, name, age, salary):
        SchoolMember.__init__(self, name, age)
        self.salary = salary 
        print('(初始化 Teacher: {})'.format(self.name))

    def tell(self):
        SchoolMember.tell(self) 
        print('Salary: {}'.format(self.salary))


# 学生类，从SchoolMember继承
class Student(SchoolMember):
    '''代表一位学生。'''

    def __init__(self, name, age, marks):
        SchoolMember.__init__(self,    name, age)
        self.marks = marks 
        print('(初始化 Student: {})'.format(self.name))

    def tell(self):
        SchoolMember.tell(self) 
        print('Marks: {}'.format(self.marks))


print('初始化过程：')
t = Teacher('张老师', 40, 10000)
s = Student('李四', 25, 75)

print('\n对象调用方法：')
# 子类对象调用各自子类方法
t.tell()
s.tell()

运行结果为：

初始化过程：
(初始化 SchoolMember: 张老师)
(初始化 Teacher: 张老师)
(初始化 SchoolMember: 李四)
(初始化 Student: 李四)

对象调用方法：
Name:张老师     Age:40          Salary: 10000
Name:李四       Age:25          Marks: 75

多态

某一老师或学生对象看作SchoolMember的对象并加以引用，这在某些情况下会大为有用，例如清点学校中的成员数量。这被称作多态性（Polymorphism），在任何情况下，如果父类型希望，子类型都可以被替换，也就是说，该对象可以被看作父类的实例。

在这里当我们使用SchoolMember类的tell方法时，可以将Teacher或Student的实例看作SchoolMember的实例。 同时，会发现被调用的是子类型的tell方法，而不是SchoolMember的tell方法。因为Python总会从当前的实际类型中开始寻找方法，如果它找不到对应的方法，它就会在该类所属的基类中依顺序逐个寻找属于基类的方法，这个基类是在定义子类时后跟的元组指定的。

不同的子类对象调用相同的父类方法，产生不同的执行结果，可以增加代码的外部调用灵活度。

  • 多态以继承和重写父类方法为前提；
  • 多态是调用方法的技巧，不会影响到类的内部设计。

将Student类也加入实现，本例为：9.6-多态.py

# 多态实现

# 公共类，基类
class SchoolMember:
    '''代表任何学校里的成员。'''

    def __init__(self, name, age):
        self.name = name
        self.age = age
        print('(初始化 SchoolMember: {})'.format(self.name))

    def tell(self):
        '''输出对象信息'''
        pass


# 老师类，从SchoolMember继承
class Teacher(SchoolMember):
    '''代表一位老师。'''

    def __init__(self, name, age, salary):
        SchoolMember.__init__(self, name, age)
        self.salary = salary 
        print('(初始化 Teacher: {})'.format(self.name))

    def tell(self):
        print('Name:{}\tAge:{}\t'.format(self.name, self.age), end='\t')
        print('Salary: {}'.format(self.salary))


# 学生类，从SchoolMember继承
class Student(SchoolMember):
    '''代表一位学生。'''

    def __init__(self, name, age, marks):
        SchoolMember.__init__(self,    name, age)
        self.marks = marks 
        print('(初始化 Student: {})'.format(self.name))

    def tell(self):
        print('Name:{}\tAge:{}\t'.format(self.name, self.age), end='\t')
        print('Marks: {}'.format(self.marks))


# 父类引用作为参数，根据传入的子类对象调用相应子类的方法
def test_tell(school_member):
    school_member.tell()


print('初始化过程：')
t = Teacher('张老师', 40, 10000)
s = Student('李四', 25, 75)

print('\n对象调用方法：')
# 子类对象调用
test_tell(t)
test_tell(s)

运行结果为：（严格来说这和强类型编程语言的多态有一定的差别，这里传入的school_member对象甚至不一定是school_member类或是它的子类，只要这个对象拥有tell方法，就能够调用。）

初始化过程：
(初始化 SchoolMember: 张老师)
(初始化 Teacher: 张老师)
(初始化 SchoolMember: 李四)
(初始化 Student: 李四)

对象调用方法：
Name:张老师     Age:40          Salary: 10000
Name:李四       Age:25          Marks: 75

如果继承元组（Inheritance Tuple）中有超过一个类，这种情况就会被称作多重继承（Multiple Inheritance）。
